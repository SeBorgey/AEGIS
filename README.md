# AEGIS
**Autonomous Engineering and Generation Intelligence System**

AEGIS — это автономная мультиагентная система для разработки программного обеспечения, специализирующаяся на создании Python-приложений с графическим интерфейсом (GUI) на базе PySide6. Система использует взаимодействие нескольких специализированных агентов для полного цикла разработки: от анализа требований до генерации кода и тестирования.

## Основные возможности

*   **Мультиагентная архитектура**:
    *   **Manager Agent**: Управляет процессом разработки, планирует задачи, создает RPD (Rapid Product Development) и контролирует работу кодера.
    *   **Coder Agent**: Отвечает за написание кода, реализацию функционала и исправление ошибок.
    *   **Judge Agent**: Выполняет black-box тестирование готовых приложений, взаимодействуя с GUI и проверяя соответствие требованиям.
*   **Генерация GUI**: Автоматическое создание современных графических интерфейсов с использованием PySide6.
*   **Веб-интерфейс**: Удобная панель управления для постановки задач (ТЗ), мониторинга процесса в реальном времени и скачивания результатов.
*   **Автономность**: Система способна самостоятельно итерировать процесс разработки, исправляя ошибки и улучшая код без участия человека.

## Установка и запуск

### Предварительные требования
*   Linux (рекомендуется)
*   Python 3.10+
*   Установленные зависимости (см. `pyproject.toml`)

### Запуск через веб-интерфейс (Рекомендуется)
Это самый простой способ взаимодействия с AEGIS.

1.  Перейдите в корневую директорию проекта.
2.  Запустите скрипт веб-интерфейса:
    ```bash
    ./web_interface.sh
    ```
    *Или используйте `run_web_interface.sh`, если он доступен.*
3.  Откройте браузер по адресу `http://localhost:8000`.
4.  Введите ваше техническое задание (ТЗ) и запустите процесс.

### Структура проекта

*   `action_api/` — API для выполнения действий агентами.
*   `web_interface/` — Исходный код веб-интерфейса (FastAPI + статика).
*   `datasets/` — Данные и примеры для обучения или контекста.
*   `manager_agent.py` — Логика управляющего агента.
*   `judge_agent.py` — Логика агента-тестировщика.
*   `react_agent.py` — Базовый класс ReAct агента.
*   `main.py` — Точка входа (CLI).
*   `runs/` — Директория, куда сохраняются результаты генерации и логи.
